本地数据

Mock&Proxy

dpack提供了一个数据mock+线上代理的组合开发模式，要做到这些，需要在项目根目录提供一个proxy.js文件，如

    module.exports = {
        '/api/v1/topics': {
            proxy: {
                target: 'https://cnodejs.org',
                changeOrigin: true
            },
            useMock: false,
            mock: {
                "success|boolean": true,
                "data": {
                    "id": /\d{10}/
                }
            }
        },
        '/api/home/hometopic': {
            proxy: {
                target: 'https://cnodejs.org',
                changeOrigin: true
            },
            useMock: true,
            mock: {
                "errno|1": [0, 20002], 
                "data|1-10": [{
                    "id": /\d{10}/,
                    "topic|1": [
                        'Hello World',
                        '你好',
                        '欢迎来到React世界',
                        '这是首页',
                        '滴滴一下'
                    ]
                }]
            }
        }
    }

export一个对象，key为api path，你可以通过useMock控制是用mock数据还是代理数据，一般而言，在开发前期，后端接口还未完成的时候，我们使用本地mock数据，等接口开发完成，我们可以修改useMock=true，采用接口数据，本地mock数据写法请参考mockjs，代理设置参考http-proxy-middleware
