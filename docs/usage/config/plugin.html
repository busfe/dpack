plugin

plugin

webpack中的plugins配置项是一个数组，其内容为webpack编译过程中所需要的各种插件。在dpack中，其为plugins内置了一些常用的webpack插件，如clean-webpack-plugin、html-webpack-plugin等等。

如果按照数组来进行配置的话，那么带来的一个问题：

覆盖其中的某个内置插件非常不方便，尤其是可以为同一个插件配置多项的时候。

为此，我们为了支持更加友好的配置，我们采用对象的形式(key-value)形式来配置webpack的组件，底层dpack会将这些plugin转化为webpack所需要的形式。如下面的配置：

    dpack.set('plugins', {
        'html-webpack-plugin': new HtmlWebpackPlugin({
                title: 'WebPack Demo',
                template: path.resolve('./src/index.html'),
                filename: 'index.html'
        })
    })

注意： 上面为dpack的plugins配置会覆盖dpack内置所有的配置插件，并且整个webpack只配置了一个插件html-webpack-plugin插件。

所以，若想覆盖dpack内置配置的插件，例如webapck.DefinePlugin插件，你可以指定对应的key(支持对象路劲字符串)。

 dpack.set('plugins.DefinePlugin', 
     new webpack.DefinePlugin({
        'process.env.NODE_ENV': JSON.stringify(args.env)
    }),
 })

另外，对于一个可能需要配置多次的webpack插件，如html-webpack-plugin或者webpack.optimize.CommonsChunkPlugin等，支持配置数组。

    dpack.set('plugins.CommonsChunkPlugin',  [ 
            new webpack.optimize.CommonsChunkPlugin({
                name: 'vendor',
                minChunks: function (module) {
                    return (
                        module.resource &&
                        /\.js$/.test(module.resource) &&
                        module.resource.indexOf(path.join(process.cwd(), 'node_modules')) === 0
                    )
                }
            }),
            new webpack.optimize.CommonsChunkPlugin({
                name: 'manifest',
                chunks: ['vendor']
            }),
        ]
    )

最后注意一点：

因为plugins采用对象形式来配置插件，所以在配置插件时，插件的key推荐使用插件的名称作为key。

如上面例子中webpack的内置插件webpack.optimize.CommonsChunkPlugin我们使用CommonsChunkPlugin作为key；而第三方提供的插件则使用其npm包的名称作为key，如html-webpack-plugin.
