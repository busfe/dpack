module

module

与配置dpack的plugins配置类似，module的loader规则配置项rules也由数组值转换为对象值来进行配置的（具体可以查看dpack的plugins配置项），易于覆盖dpack内置的配置。

dpack内置为module.rules配置了以下家几种loader。

rules.key loader description
js babel-loader 编译js、jsx文件
css css-loader 编译 css文件
less less-loader 编译 less文件
image url-loader 编译各种图片文件
font url-loader 编译font字体文件
svg url-loader 编译svg文件

使用dpack.set方法可以很容易覆盖dpack内置的loader配置规则。另外，也可以添加新的loader规则。例如添加vue-loader规则：

dpack.set('module.rules.vue', {
  test: /\.vue$/,
  loader: 'vue-loader',
  options: {
        ...
    }
})

注意点：

dpack内置了对babel-loader的相关配置，若项目中没有配置.babelrc文件则会以内置的配置来编译，否则以用户项目的配置为准。vue项目除外，vue-loader会在当前项目中找.babelrc配置，找不到编译会报错！
